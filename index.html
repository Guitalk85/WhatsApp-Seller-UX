BiofuelCircle "Next" — WhatsApp Seller UX Storyboard (v1)

Persona: Unorganized biomass manufacturer (briquette/pellet maker) Channel: WhatsApp Business API + WhatsApp Flows Scope: Happy path only · Sequential single-deal model · English only · v1 MVP

Locked Design Assumptions

  • One active deal per manufacturer at any time. New broadcasts are only pushed after current deal closes.
  • Onboarding is light and one-shot. Captures the minimum to enable broadcast targeting; richer fields (GSTIN, bank, KYC) are collected just-in-time before first invoice/payment.
  • Counter-price flow is in v1.
  • Main menu surfaces the current active deal plus past deals history.
  • Trader-side ops surface is out of scope for these mocks.

Onboarding minimums

To enable broadcast targeting, the first Flow captures:

  1. Company name
  2. Primary commodity (Soybean Briquettes / Groundnut Shell / Mustard Husk / Biomass Pellets / Mixed)
  3. Monthly capacity bucket (<100 MT / 100–500 MT / 500+ MT)
  4. District / state
  5. Years in operation

Everything else (GSTIN, bank details, lab certs, KYC) is collected progressively at the moment it becomes necessary.

Conversational state machine — happy path summary

[New number]
    → Screen 1 (Hi / recognition)
    → Screen 2 (Flow onboarding)
    → [Profile created, awaiting broadcast]

[Broadcast trigger]
    → Screen 3 (Broadcast received)
    → Screen 4 (Quantity + price decision)
        ├─ Accept → Screen 6 (dispatch)
        └─ Counter → Screen 5 (counter flow)
                    → [Trader responds]
                    → Screen 6 (dispatch)

[Dispatch loop — repeats per truck until deal balance = 0]
    → Screen 6 (vehicle + weighbridge)
    → Screen 7 (E-way + in-transit)
    → [Buyer unload trigger]
    → Screen 8 (cleared + invoice + payout)
    → [If balance > 0, loop back to Screen 6]
    → [If balance = 0, deal closes → next broadcast eligible]

[Any time]
    → Screen 9 (Main Menu) ← / → Screen 10 (Past Deals)

  • Counter-rejection branch

  • Dispute filing flow
  • Voice note input
  • Marathi/Hindi localisation
  • Trader-side ops dashboard
  • PWA companion surface
  • Re-engagement templates for inactive sellers

WABA Template Catalog — BiofuelCircle "Next" Seller Channel

Purpose: Every business-initiated outbound message from BFC to a manufacturer outside the 24-hour session window requires a Meta-pre-approved template. Below is the complete catalog needed for the v1 MVP, plus a few v1.1 templates worth submitting in parallel.

Approval lead time: 24 hours (utility) to 7+ days (marketing, ambiguous categorisation). Submit utility templates first.

Naming convention: bfc_<lifecycle>_<event>_<lang> — e.g., bfc_deal_broadcast_new_en.

Category Strategy

Meta classifies templates into three buckets, priced differently in India and approved at different bars:

  • Utility — confirmations, updates on a transaction the user initiated or is part of. Cheapest, fastest approval. Most of our catalog should land here.
  • Marketing — promotional, lead-gen, upsell. More expensive, slower approval, opt-out required.
  • Authentication — OTPs only.

Critical classification call: The "New Buying Requirement" broadcast is the borderline one. Meta could read this as marketing (an unsolicited business opportunity push) even though we'd argue it's utility (continuing an established commercial relationship the seller opted into during onboarding). Submit it as utility with a clear footer: "You receive this because you registered as a BiofuelCircle manufacturer-supplier. Reply STOP to pause." If rejected, resubmit as marketing.

Operational Notes for Dev / Ops

Session window mechanics: Every template that lands on the user opens a 24-hour free-form session. Plan flows so the user's next expected action happens within that window — otherwise the next message has to be another template. Examples: - After bfc_deal_broadcast_new_en, if the user replies Yes, interested, free-form is open for 24 hours — enough to capture quantity, price, and counter-flow. - After bfc_delivery_unloaded_settlement_en, free-form is open for 24 hours — enough to handle invoice generation. If user delays beyond that, the next prompt must use bfc_payout_early_offer_en.

Variable validation: Meta rejects templates where variables look like they could carry promotional content. Keep all {{n}} slots strictly numeric / structured data (amounts, dates, IDs, quantities, locations). Never inject free-text descriptions or marketing language.

Footer hygiene: Every marketing-leaning template needs an opt-out footer. Even utility templates benefit from a discreet "Reply HELP for support."

Rate limits: WhatsApp's tier system limits how many distinct users you can message per day. Start tier: 1,000 unique users / 24h. Tier 2 after sustained quality rating: 10,000/24h. Plan broadcast volume accordingly — for ~5,000 manufacturers on the platform, you'll need at least Tier 2.

Submission order priority: 1. bfc_deal_broadcast_new_en (highest reject risk — submit first) 2. bfc_delivery_unloaded_settlement_en (load-bearing for the flow) 3. bfc_offer_counter_accepted_en and _rejected_en 4. bfc_payout_early_offer_en (medium risk) 5. Everything else
